## ✅ 해결책

### 🥉 Level 1: 즉시 적용 (MVP)

**Semaphore로 동시 실행 제한**

```python
# app/services/background_service.py
_analysis_semaphore = asyncio.Semaphore(10)  # 최대 10개

async def analyze_photos_background(...):
    async with _analysis_semaphore:
        # 분석 로직
```

**추가 작업:**

- DB 커넥션 풀 확장: `pool_size=20, max_overflow=30`
- 타임아웃 설정: 60초
- Rate Limiter (외부 API)

**장점:**

- ✅ 구현 간단 (30분)
- ✅ 즉시 효과
- ✅ 20명 동시 접속 안정적

**적용 시기:** 동시 사용자 20명 미만

---

### 🥈 Level 2: Celery + Redis (권장)

**분산 태스크 큐 시스템**

```
FastAPI → Redis Queue → Celery Worker (10대까지 확장 가능)
```

**주요 변경:**

- Redis 설치
- Celery 설정
- 태스크 발행: `analyze_photos_task.apply_async()`
- 별도 워커 실행

**장점:**

- ✅ 수평 확장 (워커 10대+)
- ✅ 자동 재시도
- ✅ 장애 격리
- ✅ 모니터링 (Flower)

**성능:**

- 100명 동시 → 30-60초 완료
- CPU 60-70% 안정
- 메모리 2GB 안정

**적용 시기:** 동시 사용자 20-100명
